Contributions are welcome, both for documentation and for plugins/features.

## Ways to contribute

- Improve documentation (clarity, examples, translations, structure).
- Report bugs with reproducible scenarios.
- Propose or implement new steps and plugins.
- Improve existing plugin behavior, error messages, and test coverage.

## Before opening a pull request

- Verify that your change is focused and easy to review.
- Include enough context so reviewers understand the motivation.
- Keep naming and terminology consistent with existing docs and plugin conventions.

## Documentation contribution guidelines

- Keep explanations task-oriented and concrete.
- Prefer short examples that can be copied and executed quickly.
- For bilingual content, keep English and Spanish pages aligned in meaning.
- Avoid ambiguous wording; use explicit technical terms when needed.

## Code/plugin contribution guidelines

- Prefer backward-compatible changes.
- Document any new configuration key and provide at least one usage example.
- If a change affects user-facing behavior, update the corresponding docs page.

## Pull request checklist

- Scope is clear and limited.
- Docs are updated.
- New behavior is explained with examples.
- Existing behavior is not broken without explicit migration notes.
