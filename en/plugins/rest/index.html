      Configuration

      rest.baseURL

      Set the base URL for subsequents API calls. This is equivalent to the step "Define base URL" in you prefer the descriptive configuration.

      Example:

      rest:
  baseURL: https://example.org/api/v2



      rest.contentType

      Set the content type that would be sent in the request header of subsequent API calls. Accepted values are:

      literal content-type header value
      ANY */*
      TEXT text/plain
      JSON application/json, application/javascript, text/javascript, text/json
      XML application/xml, text/xml, application/xhtml+xml
      HTML text/html
      URLENC application/x-www-form-urlencoded
      BINARY application/octet-stream
      MULTIPART multipart/form-data, multipart/alternative, multipart/byteranges, multipart/digest, multipart/mixed, multipart/parallel, multipart/related, multipart/report, multipart/signed, multipart/encrypted

      Default value is JSON.

      Example:

      rest:
  contentType: XML



      rest.httpCodeThreshold

      Sets a global HTTP response code threshold. Every time an API call returns an HTTP code equals or greater, the step would automatically fail regardless any other condition.

      Default value is 500.

      Example:

      rest:
  httpCodeThreshold: 999



      rest.timeout

      Sets a response timeout (in milliseconds) for the subsequent HTTP requests. In case of exceeding the specified time, the request will be stopped and an error will occur.

      Default value is 60000.

      Example:

      rest:
  timeout: 10000



      rest.oauth2.url

      Set an OAuth 2.0 authentication service that would be used to generate a token in the request header Authorization of the API calls.

      Example:

      rest:
  oauth2: 
    url: https://accounts.google.com/o/oauth2/auth



      rest.oauth2.clientId

      Sets the parameter clientId of the OAuth 2.0 authentication service defined by the rest.oauth2.url value.

      Example:

      rest:
  oauth2: 
    clientId: WEB_APP



      rest.oauth2.clientSecret

      Sets the parameter clientSecret of the OAuth 2.0 authentication service defined by the rest.oauth2.url value.

      Example:

      rest:
  oauth2: 
    clientSecret: ABRACADABRAus1ZMGHvq9R



      rest.oauth2.cached

      Sets whether the retrieved token is cached to avoid recurring calls to the oauth service if the data is the same.

      Default value is false.

      Example:

      rest:
  oauth2:
    cached: true



      rest.oauth2.parameters

      Sets the default parameters for OAuth authentication.

      Example:

      rest:
  oauth2:
    parameters:
      grant_type: password
      username: pepe
      password: 1234asdf
      scope: something



      rest.config.multipart.subtype

      Sets the subtype of multipart HTTP requests. The available values are:

      literal
      form-data
      alternative
      byteranges
      digest
      mixed
      parallel
      related
      report
      signed
      encrypted

      Default value is form-data.

      Example:

      rest:
  config:
    multipart:
      subtype: mixed



      rest.config.multipart.filename

      Sets the filename of multipart HTTP requests.

      Default value is file.

      Example:

      rest:
  config:
    multipart:
      filename: other_name



      rest.config.redirect.follow

      Sets whether to allow following redirects in HTTP requests.

      Default value is true.

      Example:

      rest:
  config:
    redirect:
      follow: false



      rest.config.redirect.allowCircular

      Sets whether circular redirects are allowed in HTTP requests.

      Default value is false.

      Example:

      rest:
  config:
    redirect:
      allowCircular: true



      rest.config.redirect.rejectRelative

      Sets whether to reject relative redirects in HTTP requests.

      Default value is false.

      Example:

      rest:
  config:
    redirect:
      rejectRelative: true



      rest.config.redirect.max

      Sets the maximum number of redirects in HTTP requests.

      Default value is 100.

      Example:

      rest:
  config:
    redirect:
      max: 150
