Contribuir

Las contribuciones son bienvenidas, tanto en documentación como en plugins y funcionalidades.

Formas de contribuir

  • Mejorar la documentación (claridad, ejemplos, traducciones y estructura).
  • Reportar bugs con escenarios reproducibles.
  • Proponer o implementar nuevos pasos y plugins.
  • Mejorar el comportamiento de plugins existentes, sus mensajes de error y su cobertura de pruebas.

Antes de abrir una pull request

  • Verifica que el cambio esté bien acotado y sea fácil de revisar.
  • Incluye contexto suficiente para que se entienda la motivación.
  • Mantén la nomenclatura y la terminología coherentes con las convenciones actuales.

Guía para contribuciones de documentación

  • Prioriza explicaciones orientadas a tareas y con pasos concretos.
  • Usa ejemplos cortos que se puedan copiar y ejecutar rápidamente.
  • En contenido bilingüe, mantén alineado el significado entre páginas en inglés y español.
  • Evita formulaciones ambiguas y utiliza términos técnicos explícitos cuando corresponda.

Guía para contribuciones de código/plugins

  • Prioriza cambios retrocompatibles.
  • Documenta cualquier nueva clave de configuración e incluye al menos un ejemplo de uso.
  • Si el cambio afecta al comportamiento visible por usuarios, actualiza la página de documentación correspondiente.

Checklist de pull request

  • El alcance está claro y acotado.
  • La documentación está actualizada.
  • El nuevo comportamiento se explica con ejemplos.
  • El comportamiento existente no se rompe sin notas de migración explícitas.
